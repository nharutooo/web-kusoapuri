# web-kusoapuri (開発手順書)

## 開発の流れ

Issue ベースで開発を進める。

### 1. Issue を立てる

これからやる作業（機能追加やバグ修正）を GitHub の「Issues」タブに登録する。

- タイトル例: `ログイン画面の作成` `TOP画像の修正`
- 登録されると `#1` のような番号がつく。

### 2. ブランチを作成する

- VSCode の左下、またはターミナルから新しいブランチを作る。

**VSCode の左下からやる場合**

1. VSCode の左下にある `main`（または現在のブランチ名）をクリックする。
2. 上に出てくるメニューから **「＋ 新しいブランチの作成...」** を選ぶ。
3. ブランチ名を入力して Enter を押す。

**ターミナルからやる場合**

1. 次のコマンドをターミナルにうつ
   `git checkout -b feature/Issue番号-わかりやすい名前`
   (▼ 具体的な入力例)

- 例：Issue 番号が「2」で、ログイン機能を作る場合
  `git checkout -b feature/2-login` (意味：feature/2-login という新しいブランチを作って、そこに移動する)
- 例：Issue 番号が「3」で、画像のバグを直す場合
  `git checkout -b fix/3-image-fix`

**命名ルール:**

- `feature/Issue番号-内容` （例: `feature/1-add-readme-rules`）
- `/`はフォルダを作るということで、`feature/`なら feature というフォルダを作っている。また、そのフォルダ内に入れる。
- `/`の前につく`feature`や`fix`は、そのブランチが何をしているかを示す。(`feature`は機能追加。`fix`はバグの修正。他にも色々あるからそれは各々調べて欲しい。)

### 3. プッシュする

ブランチを切って作業が終わったら、変更を加えたファイルをコミットしてプッシュする。

### 4. プルリクエストを作成する

自分のコードを `main` ブランチに取り込んでもらうための「申請」を出す。

1. GitHub のリポジトリページを開く
   プッシュした直後に GitHub を開くと、画面上部に黄色い帯で **「Compare & pull request」** という緑色のボタンが表示されるから、これをクリックする。
   （※もしボタンがない場合は、「Pull requests」タブ → 「New pull request」から作成する）
2. 内容を入力する
   PR 作成画面になったら、以下の情報を入力する。
   - **Title (タイトル):** やったことの要約（コミットメッセージが自動で入ることが多い。その場合はそのままで OK）。
   - **Description (説明):** 変更内容の詳細。
     **重要:** ここに `Closes #Issue番号` と書く！（例: `Closes #2`）
     これを書くと、マージされた時に自動で Issue が「完了」になる。
3. プルリクエストを作成する
   右下の **「Create pull request」** ボタンをクリックする。

### 5. マージする

- 画面が切り替わったら、下の方にある **「Merge pull request」** ボタン（緑色）をクリックし、さらに **「Confirm merge」** を押して完了させる。
  (※マージ後、紫色の「Delete branch」ボタンが出たら、使い終わったブランチは削除して OK)
  > **💡 現場メモ:**
  > 本来の仕事では、ここでチームメンバーに「レビュー（確認）」をお願いし、承認をもらってからマージする。
  > また、レビューもマージも全てお願いする。
  > 上記どちらかが現場での主流のやり方。
  > でも、今回は練習っていうのと時間的にも、承認なしでマージできるようにしているから、セルフマージしてほしいです。

### 6. プルする

- GitHub は更新できたが、まだ自分のローカル上の main ブランチは古いままだから、GitHub からプルする。

1. main ブランチに戻る
2. ターミナルで`git pull origin main`とコマンドを打つ。

- すると GitHub のファイルの状態が全て反映されるから、自分の変更だけでなく、他の人の加えた変更も全て自分のローカル上に反映される。
- もしブランチを綺麗にしたかったら、使い終わったブランチを削除しておく。
